# P Ranker - Paper Ranking Lookup Tool

## What This Is

A client-side web app that determines the ranking (SCImago Q1-Q4 for journals, CORE A*-C for conferences) of the source venue for scientific papers given their DOIs.

## Architecture

- **Pure client-side**: HTML + CSS + vanilla JS, deployed on GitHub Pages
- **CrossRef API**: Called from browser to resolve DOIs into paper metadata
- **Pre-bundled ranking data**: `data/scimago.json` (journals) and `data/core.json` (conferences) built by Python scripts at CI time

## Key Commands

```bash
# Refresh SCImago journal data (place export as data/scimago_raw.csv or data/scimagojr*.csv)
pip install -r scripts/requirements.txt
python scripts/build_scimago.py

# Enrich core.json with DBLP topic tags (run once after build_core.py; takes ~16 min first run, cached after)
python scripts/enrich_core_dblp.py

# Find official website URLs for all conferences (cached in data/conf_urls_cache.json)
# --top processes only A*/A; omit flag for all 971 conferences (~30 min, cached after)
python scripts/find_conf_urls.py --top
# Review/edit output: data/conf_urls.csv
# Then scrape homepage descriptions into core.json:
python scripts/enrich_conf_descriptions.py --top   # A*/A only (~10 min)
python scripts/enrich_conf_descriptions.py          # all conferences (~2 hrs)

# Refresh CORE conference data
# 1. Go to https://portal.core.edu.au/conf-ranks/, select source ICORE2026, click Export CSV
# 2. Save as data/core_raw.csv
# 3. Run:
python scripts/build_core.py
# If data/core_raw.csv is present it reads from it; otherwise falls back to web scraping.

# Local dev server
python -m http.server 8000

# Lint JS
npx eslint js/app.js
```

## Project Layout

- `index.html` — app entry point (root for GitHub Pages)
- `css/style.css` — styles
- `js/app.js` — all application logic
- `data/scimago.json` — journal ISSN→quartile map (generated)
- `data/scimago_raw.csv` — SCImago raw export (source for build_scimago.py)
- `data/core.json` — conference→rank map with DBLP topics and scraped descriptions (generated)
- `data/conf_urls.csv` — conference website URLs for manual review (generated by find_conf_urls.py)
- `data/conf_urls_cache.json` — DDG search cache for find_conf_urls.py (do not edit)
- `data/conf_desc_cache.json` — scraped description cache for enrich_conf_descriptions.py (do not edit)
- `data/core_raw.csv` — CORE/ICORE raw export (source for build_core.py)
- `scripts/` — Python data build scripts (run locally or when refreshing data)
- `.github/workflows/ci.yml` — validate on PR/push
- `.github/workflows/pages.yml` — deploy to GitHub Pages (`configure-pages@v5`, `deploy-pages@v5`); repo Settings → Pages → Source: **GitHub Actions** must be set once

## Conventions

- No build step for frontend — vanilla JS only
- After changing `css/style.css` or `js/app.js`, bump the `?v=` query string on those links in `index.html` so GitHub Pages visitors get fresh assets (CDN/browser cache).
- Data JSON files are committed to the repo (generated by scripts)
- CrossRef API: always include `mailto` parameter for polite pool access
- Normalize titles to lowercase, stripped of punctuation, for fuzzy matching
